# Git Diff Fast 使用说明

## 简介

Git Diff Fast 是一个 VS Code 扩展，提供三方合并（Three-way Merge）冲突解决和分支对比功能。

## 安装

### 从 VSIX 安装

```bash
# 1. 构建 VSIX 包
npm run build

# 2. 在 VS Code 中安装
# 方法 A: 命令行
code --install-extension git-diff-0.0.1.vsix

# 方法 B: VS Code 界面
# 打开 VS Code → 扩展面板(Cmd+Shift+X) → 点击"..."菜单 → "Install from VSIX..." → 选择 .vsix 文件
```

### 开发模式调试

```bash
# 安装依赖
npm install

# 编译
npm run compile

# 按 F5 启动 Extension Development Host
```

## 功能一：冲突解决（Three-way Merge）

当 Git 仓库出现合并冲突时，使用三栏对比界面直观解决冲突。

### 打开方式

1. **状态栏**：打开包含冲突标记（`<<<<<<<` / `=======` / `>>>>>>>`）的文件时，状态栏右侧会出现 `Git Diff Fast` 按钮，点击即可打开。

2. **SCM 面板**：右键点击「Merge Changes」分组下的冲突文件 → **Resolve Conflicts**。

3. **资源管理器**：右键点击冲突文件 → **Resolve Conflicts**。

### 界面说明

```
┌─ 工具栏 ─────────────────────────────────────────────────────────────┐
│ [◀] [▶] 已解决数/总数  │  [→] [⇄] [←] [🪄]  │  粒度 ▼  │  文件名  │  [✓] [✕] │
└──────────────────────────────────────────────────────────────────────┘
┌─ Local (Yours) ──────┬─ Result ────────────────┬─ Remote (Theirs) ───┐
│                       │                         │                     │
│  你的本地修改 (只读)   │  合并结果 (可编辑)       │  对方修改 (只读)     │
│                       │                         │                     │
└───────────────────────┴─────────────────────────┴─────────────────────┘
```

**工具栏按钮：**

| 按钮 | 快捷键 | 说明 |
|------|--------|------|
| ◀ 上一个 | `F7` | 跳转到上一个冲突 |
| ▶ 下一个 | `Shift+F7` | 跳转到下一个冲突 |
| → 应用 Local | `Alt+Shift+,` | 将所有未解决冲突以 Local 版本填充 |
| ⇄ 应用双方 | — | 将所有未解决冲突以双方版本合并填充 |
| ← 应用 Remote | `Alt+Shift+.` | 将所有未解决冲突以 Remote 版本填充 |
| 🪄 智能解决 | — | 自动解决纯空白差异和纯 import 语句冲突 |

**行号槽操作：**

- 点击 Local 行号旁 `»` → 将该冲突块以 Local 版本写入 Result
- 点击 Remote 行号旁 `«` → 将该冲突块以 Remote 版本写入 Result
- 直接在 Result 编辑器手动编辑

### 操作流程

1. 打开冲突文件后，所有非冲突变更已自动合并到 Result。
2. 逐个处理带橙色背景的冲突块：点击 `»` / `«` 快速采纳某一边，或手动编辑 Result。
3. 使用 `F7` / `Shift+F7` 在冲突间导航。
4. 所有冲突解决后（状态显示 `N/0 resolved · 0 conflicts`），点击 **Accept Merge**（✓）。
5. 文件自动保存并执行 `git add`，完成暂存。

### 差异粒度

通过工具栏下拉菜单可选择字内差异粒度：

| 粒度 | 说明 |
|------|------|
| Char | 逐字符高亮差异（默认） |
| Word | 逐词高亮差异 |
| Line | 仅行级高亮，不显示字内差异 |

### 注意事项

- Accept 前会校验 Result 中无残留冲突标记，有则拒绝保存并提示。
- 关闭面板时若有未保存修改，会弹出确认对话框（Apply / Abandon / Cancel）。
- Monaco 编辑器支持完整的 Undo/Redo（`Cmd+Z` / `Cmd+Shift+Z`）。

## 功能二：分支对比（Branch Compare）

对比当前分支与目标分支（或其他 commit-ish）的文件差异。

### 打开方式

1. **资源管理器**：右键点击文件夹或任意文件 → **Compare**。
2. 在弹出的选择框中，选择目标分支 / tag / commit，或手动输入 commit-ish（如 `HEAD~3`、`abc1234`）。
3. 最近使用的 5 个引用会出现在快捷选择区。

### 界面说明

```
┌─ 工具栏 ─────────────────────────────────────────────────────────────┐
│ [↻] [⇄] │ 视图 ▼  │                     分支A ⇄ 分支B — 变更摘要     │
└──────────────────────────────────────────────────────────────────────┘
┌─ 文件树 ────────┬─ 差异视图 ─────────────────────────────────────────┐
│                 │ ┌─ 差异工具栏 ────────────────────────────────────┐ │
│ ▶ 📁 src/       │ │ [◀] [▶]  文件名  [←] [↩] │ WS ▼ │ 粒度 ▼ │ [💾] │ │
│   A file.ts     │ └─────────────────────────────────────────────────┘ │
│   M app.tsx      │ ┌─ Current ───────┬─ Target ──────────────────────┐ │
│   D old.js       │ │                 │                               │ │
│                 │ │  当前分支内容     │  目标分支内容                   │ │
│                 │ │                 │                               │ │
│                 │ └─────────────────┴───────────────────────────────┘ │
└─────────────────┴─────────────────────────────────────────────────────┘
```

**工具栏按钮：**

| 按钮 | 说明 |
|------|------|
| ↻ 刷新 | 重新获取差异数据 |
| ⇄ 反转 | 交换当前分支与目标分支 |

**差异工具栏按钮：**

| 按钮 | 快捷键 | 说明 |
|------|--------|------|
| ◀ 上一个 | `F7` | 跳转到上一个变更块 |
| ▶ 下一个 | `Shift+F7` | 跳转到下一个变更块 |
| ← 应用 hunk | — | 将当前变更块从 Target 应用到 Current（仅可编辑文件） |
| ↩ 还原文件 | — | 将当前文件还原到 HEAD 基线（仅可编辑文件） |
| 💾 保存 | — | 保存对 Current 文件的编辑（仅可编辑文件） |

**视图模式：**

| 模式 | 说明 |
|------|------|
| Tree | 按目录层级展示文件（默认） |
| Flat | 平面列表展示所有文件 |

**文件状态标识：**

| 标识 | 状态 | 说明 |
|------|------|------|
| A (绿色) | Added | 新增文件 |
| M (蓝色) | Modified | 修改文件 |
| D (红色) | Deleted | 删除文件 |
| R (紫色) | Renamed | 重命名文件 |

**忽略空白模式：**

| 模式 | 说明 |
|------|------|
| WS: None | 不忽略空白 |
| WS: Trim trailing | 忽略行尾空白 |
| WS: Collapse runs | 将连续空白折叠为单个空格 |
| WS: Ignore all | 忽略所有空白差异 |

### 操作流程

1. 在资源管理器中选择对比目标分支。
2. 在左侧文件树中点击文件查看差异。
3. 对于可编辑文件（非已删除文件），可：
   - 点击 **← 应用 hunk** 将 Target 的变更块应用到 Current
   - 点击 **↩ 还原文件** 将 Current 还原为 HEAD 基线
   - 直接编辑 Current 编辑器
   - 点击 **💾 保存** 将修改写入磁盘
